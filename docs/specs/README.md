# BeQuizzy Feature Specs

This folder stores implementation-ready specs for BeQuizzy.

## Current status
The old spec set was removed because it no longer matches this product.

Only the canonical template is being kept for now:
- `_TEMPLATE/`

## How this folder should be used
Each new BeQuizzy feature spec should be created from `_TEMPLATE/` after the product/design direction is clear.

The expected workflow is:
1. brainstorm the feature direction
2. write the feature spec
3. review the spec
4. create an implementation plan
5. implement and verify

## Likely early BeQuizzy specs
The first BeQuizzy specs will likely center on:
- candidate onboarding
- Sales/BD playbook and rubric definition
- multimodal assessment session flow
- AI scorecard generation
- evidence profile generation
- employer shortlist workflow

## Rule
Do not recreate broad bulk spec batches here.

BeQuizzy specs should be added only for product areas that are actually being built.
